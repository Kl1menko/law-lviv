# Agent Skills to use

## Навіщо вони потрібні

AI-агент повинен використовувати перевірені скіли для рев’ю коду, дизайну та архітектури, а не покладатися лише на загальні міркування.

## Базова команда

```bash
npx skills add vercel-labs/agent-skills
```

## Рекомендовані скіли

### 1. react-best-practices

Використовувати коли:

* пишеш нові React/Next.js компоненти
* оптимізуєш data fetching
* перевіряєш performance
* зменшуєш bundle size

### 2. web-design-guidelines

Використовувати коли:

* перевіряєш UI/UX
* перевіряєш accessibility
* робиш аудит форм
* перевіряєш поведінку анімацій і focus states

### 3. composition-patterns

Використовувати коли:

* проектуєш reusable components
* помічаєш забагато boolean props
* рефакториш UI library

### 4. find-skills

Використовувати коли:

* треба знайти додаткові релевантні скіли під вузьку задачу

## Як агент має їх застосовувати

* після написання важливих UI-компонентів — прогнати design/UX review
* після реалізації сторінок — перевірити performance and accessibility
* після появи дублювання у компонентах — застосувати composition review

## Важливе правило

Скіли — це допоміжний інструмент, а не привід ламати продуктову логіку, описану у `/AI`.
